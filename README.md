# skills-starter

CLI manager for AI agent skills. Manages skill discovery, installation, security scanning, and routing across multiple agents (Claude Code, Codex, Cursor, Gemini CLI, and more).

Skills are reusable markdown instructions (`SKILL.md`) that agents load on-demand, keeping your context window clean until needed.

Cross-platform: Linux, macOS, Windows (symlinks, NTFS junctions, copy fallback).

## What are Skills?

Skills are markdown files with YAML frontmatter that shape how an AI agent handles specific tasks: coding guidelines, writing styles, domain workflows. You invoke them by name, and the agent applies those rules to the current conversation.

## Features

- **`skillm` CLI** — list, install, uninstall, scan, and manage skills from the terminal
- **Library system** — clone community repos, deduplicate across sources
- **Local skill repos** — link your own skill directories for live development
- **URL install** — download single skills from URLs
- **Native agent install** — symlinks/junctions into agent directories for native discovery
- **Router** — `_router/SKILL.md` routing table for lazy-loading skills on demand
- **Agent auto-detection** — installs only into agents present on your system
- **Env var support** — respects `$CLAUDE_CONFIG_DIR`, `$CODEX_HOME` overrides
- **Security scanning** — scan skills for prompt injection and data exfiltration before installing
- **Compatible with [skills.sh](https://skills.sh)** — add community skills via `skillm add --npx <package>`
- **Plain markdown** — no build step; edit `.md` files directly

## How It Works

Two modes, use either or both:

**Native install** (recommended) — skills are linked into each agent's skills directory:

```
~/.agents/skills/my-repo/my-skill/    # source (linked skill repo)
         symlink / junction
~/.claude/skills/my-skill/            # Claude Code picks it up natively
~/.codex/skills/my-skill/             # Codex picks it up natively
```

Skills appear in `/skill-name` menu, auto-trigger based on description, and support all native features.

**Router** (lazy-loading) — skills are loaded on-demand via trigger syntax:

```
user: "$my-skill" or "skill: my-skill"
         |
~/.agents/skills/_router/     # routing table: skill-name -> path
         |
~/.agents/skills/<folder>/    # actual SKILL.md loaded into context
```

## Directory Structure

```
~/.agents/
├── skills/                   # runtime (only what you install/link)
│   ├── _router/              # routing table (auto-generated by skillm router)
│   │   └── SKILL.md
│   ├── my-skills/            # your skill repo (linked via skillm add ./path)
│   │   └── commit/
│   │       └── SKILL.md
│   └── AGENTS.md             # rules for agents using this directory
└── _library/                 # cloned third-party repos (via skillm add <url>)
    ├── vercel-labs-agent-skills/
    └── antigravity-awesome-skills/

~/ai/.../skills-starter/      # this repo (skillm source, your skills)
├── skillm.py                 # CLI manager
├── tests/
├── haz-skills/               # your skills (linked into ~/.agents/skills/)
└── README.md
```

Key: `~/.agents/skills/` contains only linked repos and the router. `~/.agents/_library/` holds cloned third-party repos, invisible to agents.

## Installation

```bash
# 1. Clone the repo anywhere
git clone https://github.com/DamianPala/skills-starter ~/path/to/skills-starter

# 2. Add skillm to PATH
ln -s ~/path/to/skills-starter/skillm.py ~/.local/bin/skillm

# 3. Create the skills directory
mkdir -p ~/.agents/skills

# 4. Link your skill repos
skillm add ~/path/to/skills-starter/my-skills

# 5. Add community repos to library
skillm add owner/repo
skillm add https://github.com/someone/skills.git

# 6. Install skills into agents
skillm install -g commit python-app

# 7. Build the router
skillm router
```

## Managing Skills

### Add skill sources

```bash
# Link a local directory (dev mode, live edits)
skillm add ./my-skills
skillm add ~/projects/company-skills

# Clone a community repo into library
skillm add owner/repo
skillm add https://github.com/someone/skills.git

# Download a single skill from URL
skillm add https://example.com/skill/my-skill.md

# Add from skills.sh ecosystem
skillm add --npx <package>
```

Local paths are symlinked into `~/.agents/skills/` (dev mode). Git repos are cloned into `~/.agents/_library/`.

### Install skills into agents

```bash
skillm install my-skill              # install into detected agents (project scope)
skillm install -g my-skill           # install globally
skillm install --from repo my-skill  # pick specific repo when name is ambiguous
skillm uninstall my-skill            # remove from agents
```

### Remove skill sources

```bash
skillm remove my-skills              # unlink from skills/ or remove from library
```

## Configuration

### 1. Grant read permissions

The agent needs permission to read files outside your project.

**Claude Code** — add to `~/.claude/settings.json`:

```json
{
  "permissions": {
    "allow": [
      "Read(~/.agents/**)"
    ]
  }
}
```

**Codex** — reads files outside the workspace by default. If you encounter permission issues, add to `~/.codex/config.toml`:

```toml
[sandbox_workspace_write]
writable_roots = ["~/.agents"]
```

### 2. Add trigger to global config

**Claude Code** — add to `~/.claude/CLAUDE.md`:

```markdown
## Custom Skills

Triggers: $skill-name, "skill: X", "load skill X".

When triggered, check ~/.agents/skills/_router/SKILL.md for the routing table.
Load the SKILL.md from the path specified there.
```

**Codex** — add the same to `~/.codex/AGENTS.md`.

## skillm CLI Reference

```bash
# Discovery
skillm list                    # list all available skills
skillm list --installed        # show installed skills with token counts
skillm list react              # filter by name or description
skillm info my-skill           # show skill details

# Install / Uninstall
skillm install -g my-skill     # install globally into all detected agents
skillm install my-skill        # install into project agents
skillm uninstall -g my-skill   # remove from agents

# Library
skillm add owner/repo          # clone into library
skillm add ./local-path        # link into skills (dev mode)
skillm add https://url/s.md    # download single skill
skillm add --npx package       # add from skills.sh
skillm remove repo-name        # remove from skills or library
skillm update                  # git pull all library repos

# Security
skillm scan my-skill           # scan for security issues
skillm scan --all              # scan everything

# Infrastructure
skillm router                  # rebuild the routing table
skillm doctor                  # diagnose broken links, missing frontmatter, etc.
skillm --version               # show version
```

## Cross-Platform

skillm works on Linux, macOS, and Windows:

| Platform | Link method | Notes |
|----------|-------------|-------|
| Linux/macOS | symlink | Default |
| Windows (Developer Mode) | symlink | Same as Unix |
| Windows (NTFS) | junction | No admin required |
| FAT32/exFAT | copy | Loses live-edit |

Fallback is automatic: symlink -> junction -> copy.
